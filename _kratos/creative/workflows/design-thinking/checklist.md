# Design Thinking Checklist

## Empathize Phase
- [ ] Users clearly identified with demographics/context
- [ ] Empathy map complete (think, feel, say, do quadrants)
- [ ] Real insights documented (not assumptions)

## Define Phase
- [ ] Point of View statement is specific and actionable
- [ ] How Might We questions generated
- [ ] Problem statement confirmed with user

## Ideate Phase
- [ ] At least 10 ideas generated before converging
- [ ] Creative technique applied systematically
- [ ] Top 2-3 ideas selected with rationale

## Prototype Phase
- [ ] Prototype definition is minimal and testable
- [ ] Scope clearly defined (what's in, what's out)

## Test Phase
- [ ] Test plan has clear success criteria
- [ ] Target testers identified
- [ ] Feedback collection method defined
